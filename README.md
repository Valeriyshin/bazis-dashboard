# Meta Ads Dashboard — TDS Media (1201997914797230)

Интерфейс анализа показателей рекламного кабинета Meta: динамика за 2 месяца,
сравнение периода к периоду и раздел письменных выводов.

## Стек
- **Next.js 15** (App Router, TypeScript) — фронт + API
- **SQLite** (better-sqlite3) — накопление снапшотов выгрузок
- **Recharts** — графики динамики
- Кастомный CSS (тёмная тема), без тяжёлых UI-зависимостей

## Разделы
- **Обзор** — KPI-карточки с дельтой + полное сравнение периодов A/B (выбор границ обоих периодов) + выбор метрик
- **Динамика** — мультивыбор метрик с наслаиванием линий (у каждой своя шкала)
- **Кампании** — дерево **Кампания → Группа → Объявление** (разворачивается), настройка столбцов, лиды/CPL/частота, KPI по лидам
- **Сводка** — 2 колонки: Главная + Деньги слева, Рекомендации справа

## Автоматическая выгрузка (этап 2)
Полная выгрузка ВСЕХ данных напрямую из Graph API (без ручного переноса):
```bash
cp .env.example .env.local     # вписать FB_ACCESS_TOKEN (ads_read)
npm run fetch                  # тянет аккаунт + все кампании/группы/объявления + лиды
npm run seed                   # пишет снапшот в БД
```
`scripts/fetch.mjs` пагинирует все уровни, считает лиды из `actions` и CPL.
Токен хранится в `.env.local` (в `.gitignore`).

## Запуск
```bash
npm install
npm run seed     # создаёт data/app.db из data/*.json
npm run dev      # http://localhost:3005
```

## Данные и БД
Схема SQLite (`data/app.db`):
- `snapshots` — метаданные каждой выгрузки (аккаунт, период, дата)
- `daily_insights` — подневные показатели аккаунта
- `campaign_insights` — показатели по кампаниям
- `summaries` — письменные выводы к снапшоту

Источник для сида — `data/account-daily.json`, `data/campaigns.json`,
`data/summary.html`. Числа уже распарсены из «сырого» формата Meta
(`1 843,91 $ (USD)` → `1843.91`).

## Процесс обновления (этап 1 — ручной)
1. Claude выгружает свежие данные из Meta через MCP-инструмент `ads_get_ad_entities`
   (метрики: spend, impressions, reach, frequency, clicks, cpc, cpm, cpp, ctr,
   actions:link_click, actions:page_engagement).
2. Обновляются `data/*.json` + пишется новая `summary.html`.
3. `npm run seed` добавляет новый снапшот (старые сохраняются — копится история).
4. Дашборд показывает последний снапшот.

> **Почему так:** MCP-инструменты Meta доступны в сессии Claude, а не как публичный
> HTTP-эндпоинт, поэтому на этапе 1 выгрузку выполняет Claude, а приложение читает
> накопленные снапшоты из SQLite.

## Ограничение кабинета
Метрики **Conversions / ROAS / Purchase недоступны** — в кабинете не настроены
purchase-события (pixel). Анализ строится на охватно-трафиковых показателях.
Конверсионные метрики подключатся автоматически, когда появятся события.
